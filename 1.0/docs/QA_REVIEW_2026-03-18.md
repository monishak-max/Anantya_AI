## Code review findings
### Improvements delivered
1. **Interpretation hierarchy**
   The prompt pack now clearly prioritizes natal promise, then dasha, then transit. This is the right direction for Anantya.

2. **Translated specificity**
   The old validator pushed the system toward leaking sign and dasha labels. The new validator is more aligned with product intent because it checks translated anchors first.

3. **Short-form context quality**
   Daily surfaces now receive more chart-specific context without forcing the frontend to become technical. This should improve relevance and reduce generic prose.

4. **Retry hardening**
   Retries now trigger on more than catastrophic failures. They also respond to low specificity, jargon leakage, em dashes, and word-count misses.

5. **Cache safety**
   Fingerprinted cache keys reduce the chance of stale or mismatched reads across members who share a name.

## Residual issues / follow-up
1. **Prompt pack still contains em dashes**
   Several prompt files still contain em dashes in instructional text or untouched examples. This does not break the code, but it slightly weakens the house-style signal because the model is still seeing em dashes in some prompts.

2. **Static QA only**
   No live LLM generation was executed in this environment. The next QA round should sample real outputs for at least:
   - chart reveal
   - now collapsed
   - now expanded
   - birth chart core

3. **Heuristic summaries are intentionally lightweight**
   The new translated bridge summaries are useful, but still heuristic. They improve grounding, not final astrological truth. The LLM must still do the real synthesis.
